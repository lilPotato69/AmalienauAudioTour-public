# Интерактивный мобильный аудиогид «Пешеходная экскурсия по Амалиенау»

Мобильное приложение для операционной системы Android, представляющее собой интерактивный аудиогид по историческому району Амалиенау (г. Калининград). Проект разработан в рамках прохождения учебной технологической (проектно-технологической) практики в ОНК «Институт высоких технологий» БФУ им. И. Канта.

Приложение сочетает в себе геоинформационную систему (отображение нити маршрута и точек интереса на базе OpenStreetMap) и отказоустойчивый мультимедийный плеер для воспроизведения экскурсионных аудиодорожек.

---

## 🛠 Стек технологий

*   **Language:** Kotlin (Coroutines)
*   **UI Framework:** Jetpack Compose (Declarative UI)
*   **Architecture:** MVVM (Model-View-ViewModel) + UDF (Unidirectional Data Flow)
*   **Geospatial / Maps:** OsmDroid (OpenStreetMap API)
*   **Data Parsing:** JSON / GeoJSON (`org.json`)
*   **Media:** Android Systems `MediaPlayer` (Audio Architecture)

---

## 🏗 Архитектурные особенности системы

Приложение спроектировано с соблюдением принципов чистой архитектуры и слабой связности компонентов:

1.  **Model (Слой данных):** Структурированные data-классы и изолированный модуль `TourParser.kt`. Парсинг тяжелых JSON и GeoJSON файлов вынесен из главного UI-потока на фоновый пул потоков `Dispatchers.IO` во избежание микрофризов интерфейса.
2.  **ViewModel (Бизнес-логика):** Класс `TourViewModel` инкапсулирует состояние экрана и управляет жизненным циклом плеера. Состояние UI транслируется в слой отображения реактивно через механизмы Compose State (`mutableStateOf`) с ограничением модификации (`private set`).
3.  **View (Слой интерфейса):** Полностью декларативный UI. Императивный компонент `MapView` из библиотеки OsmDroid корректно интегрирован в Compose-граф через связующий элемент `AndroidView`. Проведена глубокая декомпозиция интерфейса на переиспользуемые компоненты (`TourPlayerCard`, `LocateMeButton`).

## ⚙️ Ключевой функционал и решенные инженерные задачи

*   **Асинхронный конечный автомат плеера:** В модуле `AudioTourPlayer` реализована защита от состояний гонки (Race Conditions) при быстрой смене треков пользователем. За счет контроля флага `isPreparing` предотвращаются критические сбои ядра плеера (`IllegalStateException`).
*   **Динамическая кастомизация карты:** Маркеры достопримечательностей генерируются на лету с помощью Android Canvas, позволяя динамически отрисовывать номера точек и менять их визуальное состояние (выделение активного объекта) без перезагрузки карты.
*   **Runtime Permissions:** Реализован безопасный реактивный запрос системных разрешений на доступ к геолокации (`ACCESS_FINE_LOCATION`) с использованием побочных эффектов Compose.
*   **Оптимизация коллекций:** Исправлена проблема мутации внутренних коллекций OsmDroid при изменении конфигурации устройства (перевод неизменяемых списков мильстоунов в `mutableListOf`).
